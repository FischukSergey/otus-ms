# TLS Security в микросервисной архитектуре

## Текущая реализация

**Режимы SSL в PostgreSQL:**
- `disable` - SSL отключен (по умолчанию в тестах)
- `allow` - SSL используется если доступен, но не обязателен
- `prefer` - предпочитать SSL, но можно без него
- `require` - SSL обязателен, но без проверки сертификата ✅ **Используется**
- `verify-ca` - проверяет, что сертификат подписан доверенным CA
- `verify-full` - проверяет CA + hostname (CN должен совпадать с именем хоста)

**Почему выбран режим `require`:**
- ✅ Трафик действительно шифруется (защита от packet sniffing)
- ✅ Не требует управления сертификатами и CA
- ✅ PostgreSQL использует свои self-signed сертификаты


**Уровень 1 - Perimeter Security:**
- Nginx с SSL сертификатом
- Защита от внешних угроз
- Автоматическая ротация через Certbot

**Уровень 2 - Network Isolation:**
- Docker network `otus_network`
- Изоляция между контейнерами
- Нет доступа из интернета к внутренним сервисам

**Уровень 3 - Data Protection:**
- SSL соединение к PostgreSQL
- Защита самых критичных данных (credentials, user data)
- Шифрование в transit для БД
